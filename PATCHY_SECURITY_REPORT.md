### 1. server.js
**Vulnerability:** SQL_INJECTION  
**Confidence:** HIGH  
**Breaking Changes:** No

**Explanation:** 1. Removed string-interpolated queries: replaced with parameterized queries (`?` placeholders) via `pool.execute(sql, [params])` to prevent SQL injection.
2. Passwords are now hashed using bcrypt with a configurable saltRounds from environment variables; plain-text storage is eliminated.
3. Introduced `helmet` to set HTTP security headers and `express-rate-limit` to mitigate brute-force attacks.
4. Consolidated database connections into a pooled connector with `mysql2/promise`, enabling async/await usage.
5. Input presence checks ensure required fields exist before querying.


**Security Notes:** - Always keep your `dotenv` variables out of source control.
- Configure strong `SALT_ROUNDS` (12 or higher) in production to slow down brute-force cracking.
- Implement secure session management or issue signed JWTs with short expiry for authenticated users.
- Monitor and rotate database credentials periodically.
