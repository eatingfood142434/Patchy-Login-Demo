### 1. server.js
**Vulnerability:** SQL_INJECTION  
**Confidence:** HIGH  
**Breaking Changes:** Yes

**Explanation:** 1. Switched to mysql2/promise and created a connection pool using environment variables to avoid hard-coded credentials.
2. Added express-validator to enforce input validation (e.g., username alphanumeric, password length).
3. Hashed passwords with bcrypt before storage to prevent credential disclosure.
4. Replaced string interpolation in SQL with parameterized queries (.execute with ? placeholders) to prevent SQL injection.
5. Introduced JWT-based authentication: login issues short-lived tokens; protected routes use authenticateToken middleware.
6. All errors are handled gracefully without leaking stack traces to the client.

**Security Notes:** • Store JWT_SECRET and DB credentials in a secure environment or vault; rotate regularly.
• Use HTTPS in production to protect tokens in transit.
• Consider implementing refresh tokens for long-lived sessions.
• Rate-limit authentication endpoints to mitigate brute-force attacks.
• Monitor logs for suspicious activity (failed logins, SQL errors).
