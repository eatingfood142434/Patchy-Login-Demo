### 1. server.js
**Vulnerability:** SQL_INJECTION  
**Confidence:** HIGH  
**Breaking Changes:** No

**Explanation:** 1. Parameterized Queries: All SQL statements now use placeholders ('?') with parameter arrays, preventing SQL injection.
2. Password Hashing: Integrated bcrypt to hash passwords before storage and compare hashed values on login, removing plaintext password storage.
3. Authentication: Added JWT-based authentication middleware (authenticateToken) to protect routes and prevent unauthorized access.
4. Input Validation: Leveraged express-validator to enforce allowlists on username/password, mitigating malformed or malicious inputs.
5. Configuration: Moved secrets (DB credentials, JWT secret) into environment variables via dotenv, avoiding hardcoded credentials.

**Security Notes:** – Use a strong, random JWT_SECRET and rotate periodically.
– Consider rate limiting or CAPTCHA to prevent brute-force login attempts.
– Ensure SSL/TLS is enforced for all client-server communications.
– Monitor and log authentication failures and suspicious activities.
– Adjust bcrypt salt rounds based on performance/security tradeoff.
