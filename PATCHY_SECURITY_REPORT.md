### 1. server.js
**Vulnerability:** SQL_INJECTION  
**Confidence:** HIGH  
**Breaking Changes:** Yes

**Explanation:** 1. Replaced plaintext/internal memory DB with a real MySQL connection pool (mysql2/promise).
2. All SQL statements now use prepared statements with placeholders (?) to prevent SQL injection.
3. Passwords are hashed with bcrypt before storage; login compares via bcrypt.compare.
4. Introduced express-session with secure cookie settings to manage sessions (httpOnly, secure in production).
5. Added input validation via express-validator on /register and /login endpoints (all fields validated).
6. Protect /data and /logout via requireAuth middleware to enforce authentication.
7. Regenerated sessions on login to prevent session fixation.
8. Added Helmet for basic HTTP header hardening.
9. Error handling standardized to avoid internal data leakage.

**Security Notes:** • Ensure .env file is kept out of version control and has strong, random SESSION_SECRET and DB credentials.
• Use HTTPS in production to secure cookies and transport.
• Monitor and log authentication failures and rate-limit login attempts.
• Regularly rotate session secrets and database credentials.
• Consider CSP and additional headers via Helmet as needed.
• For horizontal authorization, ensure user_id scoping to avoid data leakage between users.
